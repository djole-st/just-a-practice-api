Just a random API made as a practice

Prologue

Dear visitor, welcome to the API i made as a practice when i had a bit of spare time.
I suggest using Postman if possible to send diferent requests and view their response. Here is link on how to send various requests and use it.
This is not a full project version, but worry not, you can find it in one of my GitHub repositories.
Full version has e-mail sending available, but use it carefully when you use it as you need to set security level of a account which sends e-mail to a lower level.
Check out my LinkedIn also if you are interested.
Wish you all the best and have fun :D !!!

MethodEndpointDetailsAuthorised
GET / Front page No
POST /api/users/authorise User Authorisation No
POST /api/users/authorised Authorised Personel Only YES
GET /api/users/list JSON Array of User Objects No
GET /api/users/get/1 JSON Object of User with id 1 No
GET /api/users/get?id=1 JSON Object of User with id 1 No
GET /api/users/delete?id=1 Delete User with id 1 Yes
DELETE /api/users/delete/1 Delete User with id 1 Yes
POST /api/users/add Adds new user No
POST /api/users/update/email Updates email of user No
POST /api/users/update/password Updates password of user No

POST: /api/users/add

Request Example

{"username":"test_user","password":"test_password","email":"testemail@gmail.com"}

Username can only contain lowercase/uppercase letters and ., -, _.(No numbers and special characters)

Response Example

Response: Success/Error message

Succesfully added or Username must not contain numbers/special characters.

POST: /api/users/update/password

Request Example

{"username": "test_username","password": "test_password","new_password": "new_test_password"}

Updates users password to "new_password"

Response Example

Response: Success/Error message

Succesfully updated or User not found.

POST: /api/users/update/email

Request Example

Response Example

Response: Success/Error message
